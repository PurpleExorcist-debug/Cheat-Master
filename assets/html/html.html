HTML 5 AVANZADO Y FUNDAMENTOS

Fundamentos esenciales y trucos para la estructura semántica y la accesibilidad.

Estructura Mínima de un Documento HTML5

La base de todo documento web. Es el punto de partida para cualquier proyecto y define las secciones principales (`head` y `body`).

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Título Aquí</title>
</head>
<body>
    <!-- Contenido visible para el usuario -->
    <h1>Hola Mundo</h1>
</body>
</html>
Etiquetas Clave:
  • **<!DOCTYPE html>**: Define el documento como HTML5.
  • **<html lang="es">**: Contenedor principal. El atributo `lang` es crucial para la accesibilidad (lector de pantalla).
  • **<head>**: Contiene metadatos (información no visible).
  • **<meta charset="UTF-8">**: Soporte para la mayoría de los caracteres, incluyendo acentos.
  • **<meta name="viewport">**: Esencial para el diseño responsivo (móviles).
  • **<body>**: Contiene todo el contenido visible de la página.
Cómo Vincular CSS Externo

La mejor práctica es vincular el CSS externamente en el `` para que los estilos se carguen y apliquen antes de que el navegador muestre el contenido, evitando el parpadeo de contenido sin estilo (FOUC).

<head>
    <meta charset="UTF-8">
    <title>...</title>
    
    <link rel="stylesheet" href="styles/mi-estilo.css">
    
    <link rel="stylesheet" href="styles/componentes.css">

</head>
Atributos Clave:
  • **<link>**: La etiqueta utilizada para vincular recursos externos.
  • **rel="stylesheet"**: Define la relación: este enlace es una hoja de estilos.
  • **href="..."**: La ruta donde se encuentra tu archivo CSS (puede ser relativa o absoluta).
Imágenes Responsivas con <picture>

Permite al navegador cargar la imagen más optimizada según el tamaño de la pantalla o el formato soportado, mejorando la performance.

<picture>
  <source srcset="imagen-grande.webp" media="(min-width: 800px)" type="image/webp">
  <source srcset="imagen-pequena.jpg" type="image/jpeg">
  <img src="imagen-default.jpg" alt="Descripción de la imagen" class="img-fluid">
</picture>
Etiquetas Clave:
  • **<picture>**: Contenedor que permite definir múltiples versiones de una imagen.
  • **<source>**: Define la URL de la imagen (`srcset`), la condición para usarla (`media`), y el tipo de archivo (`type`).
  • **<img>**: Es el *fallback* obligatorio; se usa si ninguna <source> coincide o es soportada.
Carga Asíncrona de Scripts

Evita que la carga de archivos JavaScript pesados bloquee el renderizado de la página (el usuario verá el contenido más rápido).

<script src="analiticas.js" async></script>
<script src="interactividad.js" defer></script>
Atributos Clave:
  • **async**: Se carga y ejecuta tan pronto como esté listo. Ideal para scripts independientes.
  • **defer**: Se carga en paralelo y se ejecuta después de que el HTML haya sido completamente parseado. Ideal para scripts que manipulan el DOM.
Contenido Distribuible con <article>

Se usa para contenido que es completo e independiente (blog, comentario, producto). Si se saca de la página, sigue teniendo sentido.

<article class="post-blog">
  <header>
    <h2>Título del Post</h2>
    <p>Publicado por: <time datetime="2023-10-26">26 Oct 2023</time></p>
  </header>
  <p>Contenido principal del artículo...</p>
  <footer>
    <p><a href="#comments">Ver 5 comentarios</a></p>
  </footer>
</article>
Etiquetas Clave:
  • **<article>**: Contenedor para contenido autosuficiente.
  • **<time>**: Muestra una fecha/hora legible y una fecha en formato máquina (atributo `datetime`).
Notificaciones de Accesibilidad con `aria-live`

Informa a los lectores de pantalla sobre cambios importantes o errores en la página (ej. un mensaje de éxito/error) sin refrescar la página.

<div role="status" aria-live="polite" class="visually-hidden">
  El formulario se ha enviado correctamente.
</div>
Atributos Clave:
  • **aria-live="polite"**: Indica que los cambios en este elemento deben ser anunciados por el lector de pantalla sin interrumpir al usuario.
  • **role="status"**: Semánticamente identifica el elemento como un contenedor de mensajes de estado.
Listas de Definición (<dl>)

Perfecto para mostrar pares de nombres y valores (metadata, glosarios, preguntas frecuentes).

<dl>
  <dt>Framework</dt>
  <dd>Bootstrap 5</dd>
  
  <dt>Colores</dt>
  <dd>Futuristic Palette (Dark Mode)</dd>
  
  <dt>Tipografía</dt>
  <dd>Nunito (General) / Fira Code (Código)</dd>
</dl>
Etiquetas Clave:
  • **<dl>**: Contenedor principal de la lista de definiciones.
  • **<dt>**: Define el término.
  • **<dd>**: Define la descripción/valor.
Validación de Entrada con `pattern`

Usa expresiones regulares para validar automáticamente el formato de un campo antes de enviar el formulario, sin JavaScript.

<input 
  type="text" 
  name="zip-code" 
  required 
  pattern="[0-9]{5}" 
  title="El código postal debe tener 5 dígitos (0-9).">

<input 
  type="text" 
  name="username" 
  pattern="[a-zA-Z0-9]{3,15}"
  title="Solo letras y números, de 3 a 15 caracteres.">
Atributos Clave:
  • **pattern**: Recibe una expresión regular.
  • **title**: El mensaje de error que se muestra al usuario si el patrón no se cumple.
Open Graph (OG) Metatags

Define cómo se ve tu enlace cuando se comparte en redes sociales (Facebook, LinkedIn, WhatsApp). Es esencial para el marketing.

<meta property="og:title" content="CHEAT MASTER: HTML Avanzado">
<meta property="og:description" content="Tu arsenal de trucos de código HTML semántico y performance.">
<meta property="og:image" content="https://tudominio.com/img/html-share.jpg">
<meta property="og:url" content="https://tudominio.com/html.html">
<meta property="og:type" content="website">
    
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:site" content="@tuusuario">
Atributos Clave:
  • **property="og:..."**: Prefijo estándar para Open Graph (OG).
  • **og:image**: La URL absoluta de la imagen que se mostrará.
Plantillas Reutilizables con <template>

Define un fragmento de HTML que no se renderiza en la página inicialmente, pero puede ser clonado e inyectado con JavaScript posteriormente.

<template id="plantilla-producto">
  <div class="card-producto">
    <h3 class="nombre"></h3>
    <p class="precio"></p>
    <button>Agregar</button>
  </div>
</template>
Etiquetas Clave:
  • **<template>**: Contenedor para código HTML que se guarda como borrador.
  • **id**: Esencial para que JavaScript pueda acceder a la plantilla.
